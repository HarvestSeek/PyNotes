    1.Module

    1. import

    每个模块包含一些函数定义和一部分可执行代码。如果当前文件想使用另一个 py 模块，可以使用 import 语句： 

    1
2
3
    import sys

print(sys.path)
    其格式为 import <module_name> 或 import <module_name> as <bias>。
    • 在导入时，每个模块内的脚本均会被执行一次；
    • 不管执行多少次 import，每个模块只会被执行一次；
    • python 解释器在搜索路径 (sys.path) 中寻找指定的模块。搜索目录包括：当前目录、python 安装时确定的系统路径、库安装时确定的路径。

     使用 import，源模块的命名空间 (变量名、函数名) 并没有导入当前模块，如果需要访问变量或使用函数，需要通过 module.<name> 的形式访问。
     如果某个函数或变量需要被经常使用，可以为它赋一个本地名称： 

    1
2
3
4
    import sys

gettrace = sys.gettrace
print(gettrace())

    2. from ... import

    可以通过 from ... import 导入指定的部分到当前的符号表中： 

    1
    from <module_name> import <arg_name1>, <arg_name2>, <func_name1>, ...
    但这种方法不会把 <module_name> 导入到当前的符号表。

    或者直接导入全部内容 (除了以单一下划线开头的名字) 

    1
    from <module_name> import *
    这种方法并不建议频繁使用，因为它存在覆盖已有定义的风险。

    3. __name__ 属性

     Python 中每个模块都有一个 __name__ 属性。只有当 py 文件被当作脚本执行而不是被当作模块引入的时候，__name__ 的值才等于 '__main__'。
     可以利用这个特性运行测试。

    1
2
3
    ...
if __name__ == '__main__':
    # do the testing ...

    4. Package

     Python 也通过包管理一个项目不同模块的命名空间。
     一个目录只有在包含一个 __init.py__ 的时候才会被认为是 Python 的一个 Package。在导入一个包的时候，首先会默认执行 __init.py__ 里的代码，因此可以利用 __init.py__ 做一些初始化的工作，例如导入一些默认模块。

    (1). 导入方法

    1
2
3
    import item.subitem.subsubitem

item.subitem.subsubitem.func()
     以上面的代码为例，如果是通过 import 导入，前面的每一项必须都是包，而最后一项 (subsubitem) 可以是包名或模块名 (py 文件名)，并且通过这种方式导入的模块中的变量和函数必须通过全名使用。

    1
    from package import item
     而如果是通过上面的方式导入，item 可以是子包、模块、函数或变量的名字 (不能包含 ".")。
     需要注意的是，以上两种方法如果最终导入的是一个包，只是相当于执行相应包中的 __init.py__，而没有导入相应包中的子模块。

    """
example:
.
└── mainpackage
    ├── __init__.py
    ├── s0.py
    ├── subpackage1
    |   ├── __init__.py
    │   └── s1.py
    └── subpackage2
        ├── __init__.py
        ├── s2.py
        └── s3.py
└── demo.py
"""

# in demo.py:
import mainpackage.subpackage1

# invalid:
mainpackage.subpackage1.s1.func()
    在上面的例子中，只有 subpackage1 的 __init.py__ 中导入了 s1，s1 符号表才能成功添加到 demo.py 中，上面的调用语句才能生效。

    (2). from <package_name> import *

     由于 Windows 是一个不区分大小写的系统，因此，在使用 from <package_name> import * 时默认不会导入任何子模块，也只是相当于 (1) 中的导入 __init.py__ 而已。
     实际上，from <package_name> import * 会导入 <package_name> 的 __init.py__ 中的列表变量 __all__ 内的所有名字。
     对于 (1) 中的最后一个例子，mainpackage 可以这样设置： 

    1
2
3
    # __init.py__ of mainpackage

__all__ = ['subpackage1', 'subpackage2', 's0']
    尽管这种导入方法很方便，但仍然推荐 from <package_name> import <specific_module>

    (3). 同级导入

    沿用 (1) 中的例子，可以通过以下方式避免使用绝对路径导入： 

    1
2
    # file: s2.py
from . import s3 # eql: from mainpackage.subpackage2 import s3
