    1. 继承

    Python 通过以下方式实现继承 (即获得父类的属性和方法)：

    2. Duck Typing

     "鸭子类型" 源于 India 诗人 Riley 的一句诗，在程序设计中，它指的是动态类型语言和某些静态语言的一种对象推断的风格。在 Python 中的体现是：

    • 变量绑定的类型是不确定的，函数和方法可以接受任何类型的参数；
    • 调用是否成功由参数的方法和属性而不是类型确定，调用不成功则抛出错误。

    3. 多继承与 MixIn

    Python 支持多继承

    class Shooter(object):

    ad = 0

    def increase_ad(self):
        self.ad += 1

class Teemo(Mage, Shooter):
    ...
     MixIn 是一种特殊的多重继承，即利用多重继承实现为一个类扩展功能。
     以多层继承问题为例，当一个类的变化有多个维度，为了避免过多的继承，就可以使用 MixIn 类作为功能模块进行扩展 (另一种办法是使用桥接模式)： 
    1
2
3
4
5
6
    class RunnableMixIn(object):
    def run():
        print("running")

class Teemo(Mage, Shooter, RunnableMixIn):
    ...
    MixIn 类的特性总结起来就是：
    • 不能生成实例，属于抽象类；
    • 不能继承 MixIn 类以外的类。
    这与 Java 中的 Interface 很相似。 不同的是，Java 的接口只提供 "规格" 上的多重继承，而 MixIn 类同时提供 "规格" 和 "实现" 上的多重继承，使用上更方便。
