    3. 字符编码

    • ASCII: 早期计算机被发明时的字符编码，包含英文字母、数字、符号等 127 个字符，每个占 1 字节
    • Unicode: 统一了各种语言 (GBK ...) 和符号的编码，解决了乱码问题。一般每个字符占 2 个字节，部分生僻字符占 4 个字节以上。
    • UTF-8: 如果文本中英文字符占多数，Unicode 比较浪费存储空间，因此实现了可变长的 UTF-8 编码。它根据 Unicode 的长度进行编码：英文字符被编码为 1 个字节，汉字通常是 3 个字节。一个 Unicode 码可能被转换成 1$\sim$6 个 Byte 的 UTF-8 码，取决于 Unicode 码的长度。由于 UTF-8 的编码规则，它也是 ASCII 码的一个超集。通常，存储或通信选择对相应字符集节省空间的编码：
      • 内存中使用 Unicode 编码，磁盘中使用 UTF-8 编码
      • 网络通信使用 UTF-8 编码

    # create by Constructor
s1 = bytes() 
s2 = bytes('hello', encoding = 'ascii')

# create by declaration
# note: if the character can't be denoted by ASCII, we denote 
#  it by format '/x##'
s3 = b'a'      # default chars: ascii

     在写入文件或进行网络传输时，需要把字符串通过 encode() 存储在 bytes 中，当读取的时候再通过 decode() 解码。
     为避免乱码问题，encode() 的参数应尽可能使用 "utf-8"。

    (2). py 文件的编码

    py 文件的开头通常这样开头： 

    1
2
    #!/usr/bin/env python3
# -*- coding: utf-8 -*-
     第一行的目的是告诉 Linux 系统，这是一个 Python 可执行程序，Windows 会默认忽略这一条。
     第二行的目的是告诉 Python 解释器，按照 utf-8 编码读取源代码。只有文本编辑器设为 utf-8 并且加上这个开头，命令行测试的时候才一定会正常显示中文。
